# Phase 2 - Blueprint Semantic Validation (v3)

## Purpose

This phase validates the semantic design produced in Phase 1 using both minimal and realistic scenarios.

---

## Scenario layers

### Minimal scenario

- product modeling
- inventory inbound
- sales order

### Realistic scenario

- fruit wholesale
- mixed measurement units
- reservation vs actual sale
- daily stock count and adjustment

---

## Key evolution

Compared to v2, this version explicitly introduces a realistic ERP-like scenario to stress test:

- modeling patterns under pressure
- semantic vs orchestration boundary
- quantity normalization and unit conversion
